📘 Frontend Implementation Document
Secure Hybrid Cloud B2B Electronics Ordering System
1️⃣ Frontend Overview
🎯 Purpose

The frontend application provides:

Company registration & login

Secure JWT-based authentication

Order creation interface

Order tracking dashboard

Real-time order status monitoring

It communicates only with the Public Cloud API and never directly with private services.

2️⃣ Architecture Overview
Frontend (React)
        ↓

        ↓

        ↓

        ↓

Key Principles:

Frontend never accesses private network.

All authentication handled via JWT.

Order status updates pulled from backend via polling.

Secure separation of concerns.

5️⃣ Authentication Flow
🔐 Login Process

User enters email & password

POST /login

Backend returns:

token
user info

Store token in localStorage

Attach token to all future API requests

Axios Setup
import axios from "axios";

const api = axios.create({
  baseURL: "http://localhost:3000"
});

api.interceptors.request.use(config => {
  const token = localStorage.getItem("token");
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

export default api;

8️⃣ Real-Time Status Updates

Since backend is async (RabbitMQ), frontend should:

Option A — Polling (Simple & Practical)

Every 5 seconds:

useEffect(() => {
  const interval = setInterval(() => {
    fetchOrders();
  }, 5000);

  return () => clearInterval(interval);
}, []);

This ensures UI updates automatically when status changes.
